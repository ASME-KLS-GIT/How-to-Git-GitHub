WHAT IS GITHUB?
===============

GitHub is a cloud-based platform that hosts Git repositories online. While Git is
the tool you use on your own computer to track changes, GitHub is the place where
you store those repositories on the internet — making it easy to back up your work,
share it with others, and collaborate on projects from anywhere in the world.

Think of it this way: Git is the technology, GitHub is the service built on top of it.

GitHub was founded in 2008 and acquired by Microsoft in 2018. Today it is home to
over 100 million developers and hundreds of millions of repositories, making it the
largest code hosting platform on the planet.

WHAT GITHUB ADDS ON TOP OF GIT
-------------------------------
Git alone is a command-line tool. GitHub wraps it in a visual, web-based interface
and adds a rich set of collaboration features:

  - Remote Repositories: Store your Git repo online so it's accessible from any
    device, and safe even if your computer is lost or damaged.

  - Pull Requests: A way to propose changes to a project. Others can review your
    code, leave comments, and approve or reject the changes before they are merged.

  - Issues: A built-in tracker for bugs, feature requests, and tasks. Great for
    organising work on a project.

  - Forks: A personal copy of someone else's repository. You can experiment with
    changes freely and then propose them back to the original via a pull request.

  - GitHub Actions: Automation workflows — for example, automatically running tests
    every time code is pushed to the repository.

  - GitHub Pages: Free hosting for static websites directly from a repository.

KEY CONCEPTS
------------
  - Remote: A version of your repository stored on GitHub (as opposed to the local
    copy on your computer).

  - Push: Uploading your local commits to GitHub.

  - Pull: Downloading the latest changes from GitHub to your local machine.

  - Clone: Creating a local copy of a repository that already exists on GitHub.

  - Fork: Copying someone else's repository to your own GitHub account so you can
    modify it independently.

  - Star: Bookmarking a repository you find interesting — similar to a "like".

WHY USE GITHUB?
---------------
  - Back up your code safely in the cloud.
  - Collaborate with teammates or contributors anywhere in the world.
  - Showcase your projects — your GitHub profile acts as a portfolio.
  - Contribute to open-source projects used by millions.
  - Automate repetitive tasks like testing and deployment.

GitHub is not just for developers. Writers use it to collaborate on documents,
researchers use it to share data, and designers use it to version their work.
