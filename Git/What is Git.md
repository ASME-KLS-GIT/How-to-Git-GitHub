WHAT IS GIT?
============

Git is a free and open-source version control system — a tool that tracks changes
in your files over time. It was created by Linus Torvalds in 2005 (the same person
who created Linux) and has since become the most widely used version control system
in the world.

At its core, Git helps you answer questions like:
  - What changed in my project, and when?
  - Who made this change?
  - Can I go back to an older version of my work?
  - Can multiple people work on the same project without overwriting each other?

HOW IT WORKS
------------
Git works by taking "snapshots" of your project at different points in time. Each
snapshot is called a commit. You can move between commits, create separate branches
to experiment with new ideas, and merge those branches back together when you're
ready — all without risking your main work.

Git runs locally on your computer, meaning you don't need an internet connection to
use it. Everything — your history, your branches, your commits — lives right on
your machine.

KEY CONCEPTS
------------
  - Repository (Repo): A folder tracked by Git. It stores all your files and their
    full history.

  - Commit: A saved snapshot of your project at a specific moment. Think of it like
    a checkpoint in a video game.

  - Branch: A separate line of development. You can create a branch to try something
    new without affecting the main project.

  - Merge: Combining the changes from one branch into another.

  - Staging Area: A space where you prepare changes before committing them. You
    choose exactly what goes into each snapshot.

WHY USE GIT?
------------
  - Never lose your work — you can always roll back to a previous version.
  - Collaborate safely — multiple people can work on the same project simultaneously.
  - Experiment freely — branches let you try new things without breaking anything.
  - Keep a clear history — every change is logged with a message and timestamp.

Git is used by developers, writers, designers, data scientists, and anyone who works
with files that change over time.
